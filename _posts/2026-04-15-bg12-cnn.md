Convolutional neural networks exploit spatial structure in data through local connectivity, weight sharing, and hierarchical feature extraction.

## Convolution Operation

A 2D convolution with kernel $K$ of size $k \times k$ applied to input $I$:

$$
(I * K)_{ij} = \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} I_{i+m, j+n} \cdot K_{m,n}
$$

Key properties:
- **Translation equivariance**: if the input shifts, the output shifts by the same amount
- **Parameter sharing**: the same kernel is applied across all spatial locations
- **Local connectivity**: each output depends only on a small receptive field

## CNN Architecture

A typical CNN alternates:

1. **Convolution layers**: extract features with learnable filters
2. **Activation** (ReLU): introduce non-linearity
3. **Pooling** (max or average): reduce spatial dimensions, increase invariance

Output dimensions for a conv layer with input size $W$, kernel $k$, stride $s$, padding $p$:

$$
W_{\text{out}} = \frac{W - k + 2p}{s} + 1
$$

## Feature Hierarchy

Lower layers learn **low-level features** (edges, textures), deeper layers learn **high-level features** (parts, objects). This hierarchical representation is what makes deep CNN so powerful.

## Residual Networks (ResNets)

**Residual connections** solve the degradation problem in very deep networks:

$$
\mathbf{h}^{(l+1)} = \mathbf{h}^{(l)} + F(\mathbf{h}^{(l)})
$$

The network only needs to learn the **residual** $F(\mathbf{h})$. If $F = 0$, the layer becomes an identity — information flows freely through skip connections.

This enabled training networks with 100+ layers (ResNet-152) and remains the backbone of most vision architectures.

## Transfer Learning

Pre-trained CNN features generalize across tasks:
1. Train a CNN on a large dataset (e.g., ImageNet)
2. Freeze early layers (generic features)
3. Fine-tune later layers on the target task

This is why CNNs are practical even with small datasets — the learned feature hierarchy transfers effectively.
